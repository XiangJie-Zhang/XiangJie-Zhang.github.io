开窗函数简介

开窗函数定义

所谓开窗函数就是定义一个行为列。

简单讲，就是在你查询的结果上，直接多出一列值（可以是聚合值或是排序号），特征就是带有over（）。

就是为每一行开窗(指定范围)，在窗内处理范围内数据，并输出在这一行。

开窗函数里的”窗”，即”窗口”，表示分析函数分析时要处理的数据范围；比如分析函数sum()，它所针对的数据范围为窗口中的记录，而不是整个表的记录，要获取某个范围的sum值，则需在窗口指定具体的数据范围，比如指定该窗口从该分组中的第一行到最后一行,那么该组中的每一个sum值都会一样，即整个组的总和

函数执行时机

FROM，JOIN，WHERE，GROUP BY，HAVING，开窗，ORDER BY，LIMIT，SELECT，DISTINCT

换言之，执行时机为，已经确定好查询的数据后，分析这些数据

函数使用位置/规范

select 关键字后/子句

分析函数() over(partition by xxx order by yyy rows between zzz and zzz)

开窗函数就是over()，over()指定范围数据分区或者排序。

分析函数是只一些对窗口内的数据处理后返回一行的数据，可以是：排序类(rank、row_number)、sum、count、avg

然后分析函数对over()函数设置的数据开始计算。

例如：

OVER ( ORDER BY fsalary DESC )对所有数据根据fsalary降序排序后，然后使用ROW_NUMBER ( )来为每一条数据添加伪列。

窗口子句(rows)的相关关键字：

  • preceding：表示在…之前
    1. 1 preceding：表示当前记录的前1条记录
    2. n preceding：表示当前记录的前n条记录
    3. unbounded preceding：不受控制的，无限的
      • 若无分组，则表示所有记录的第1条记录
      • 若有分组，则表示分组后，组内的第1条记录
  • following：表示在…之后
    1. 1 following：表示当前记录的后一条记录
    2. n following：表示当前记录的后n条记录
    3. unbounded following：不受控制的，无限的
      • 若无分组，则表示所有记录的最后一条记录
      • 若有分组，则表示分组后，组内的最后一条记录

样例

  1. rows between unbounded preceding and unbounded following：针对所有记录
  2. rows between unbounded preceding and current row：针对第一条记录到当前记录
  3. rows between current row and unbounded following：针对当前记录到最后一条记录
  4. rows between 1 preceding and current row：针对当前记录的上一行记录到当前行记录
  5. rows between current row and 3 following：针对当前记录到当前行记录的后三条记录
  6. rows between 1 preceding and 2 following：针对当前记录的上一条记录 ~~ 当前行记录的后两条记录
  7. 当开窗函数over()出现分组(partition by)子句时，unbounded preceding即表中一个分组里的第一行，unbounded following即表中一个分组里的最后一行
  8. 当开窗函数over()省略了分组(partition by)子句时，unbounded preceding即表中的第一行， unbounded following即表中的最后一行

与聚合函数的区别

聚合函数将多行数据聚合成一行。

开窗函数输入行=输出行。

因此，使用开窗函数，可能要时常伴有去重distinct。

支持的数据库

Oracle、SQL Server、DB2、PostgreSQL。

开窗函数分类

根据使用的目的，开窗函数可以分为：排序开窗函数、最值函数、偏移函数

三种常用的排序开窗函数:

  1. ROW_NUMBER() OVER () ;
  2. RANK() OVER ();
  3. DENSE_RANK() OVER () ;

常用的分析函数取首值和尾值：

  1. FIRST_VALUE();
  2. LAST_VALUE ();

用在开窗函数外：例如，对开窗函数的数据取特定行：FIRST_VALUE ( english ) OVER ( PARTITION BY departmentid, classid ORDER BY english DESC ) AS sort

常用的偏移函数：

  1. LAG(filed, num, defaultValue)：filed需要查找的字段；num往前查找的第num行的数据；defaultValue没有符合条件的默认值。
  2. LEAG(filed, num, defaultValue)：filed需要查找的字段；num往后查找的第num行的数据；defaultValue没有符合条件的默认值。

用在开窗函数外：例如，对开窗函数的数据中这一列延后展示一行：LEAG ( english,1,NULL ) OVER ( PARTITION BY departmentid, classid ORDER BY english DESC ) AS sort

如果想在开窗内指定某些行处理，则需要在开窗函数over()中使用rows between ，指定某些行

排序+开窗函数

排序开窗函数有三种，其中的不同在于：

  1. ROW_NUMBER ( )：生成唯一的序号，值相同，序号也不同
  2. RANK ( )：当值相同时，序号相同，跳过之前本应该有的值 ，例如： 5 5 7
  3. DENSE_RANK() ：当值相同时，序号相同 ，例如：5 5 6

首尾值函数+开窗函数

FIRST_VALUE()：获取窗口内第一个值

LAST_VALUE()：获取窗口内最后一个值

举个例子，获取到每个department、class中的英语分数最高分

分组+开窗函数(hive)

对开窗函数的数据(有序的)之后进行分组操作。

NTILE(n)：把有序分区中的行分发到指定数据的组中，各个组有编号，编号从1开始。n必须为int类型。

处理场景：

对于某一列的数据，展示百分比的内容。例如：查询前20%时间的订单信息。

即，对时间一列进行开窗排序后，使用ntile(5)分为5组，这样第一组的数据就是前20%！！！

偏移量+开窗函数

LAG(col,n,default_val)：往前第n行数据

LEAD(col,n, default_val)：往后第n行数据

对开窗函数处理好的数据(窗内数据)，展示本行的前/后几行数据。

处理场景：

查看顾客上次的购买时间：首先对顾客分区，时间排序。然后分析函数采用LAG展示前一行。

什么时候该用开窗函数

平常我们使用 hive或者 mysql时，一般聚合函数用的比较多。但对于某些偏分析的需求，group by可能很费力，子查询很多，这个时候就需要使用窗口分析函数了~

例如：上述我们要求每个部英语最高分和每个班数学最高分，用group by不能同时计算
