类模板

Scala类特点：

  1. Scala代码中的属性必须显式的赋值其初始值，如果不确定赋予什么值用_代替，由JVM操作
  2. var代表变量，val代表常量
  3. 属性，方法默认都是public权限，但是Scala没有public关键字
  4. 如果依据Java Bean代码规范形成Setter、Getter；则需要对属性加上注解@BeanProperty

包

Scala包的作用和Java一致：

  • 区分相同名字的类
  • 当类很多时,可以很好的管理类
  • 控制访问范围

但是，Scala的包能实现如下操作，而Java不可：

  1. Scala中包名和源码所在的系统文件目录结构要可以不一致，但是编译后的字节码文件路径和包名会保持一致(这个工作由编译器完成)。

  2. Package可以在同一源码中使用多次，如果类内容没有放在package块中，则类在最后的那个包中

  3. 包有作用域，子包可以直接使用父包的内容

  4. Package可以是对象，此时能在其中放入除了伴生类/伴生对象之外的方法/变量

  5. 包名可以相对路径也可以绝对路径

import

  1. import用于导入类

  2. import可以用在任意的地方

  3. import导入一个包的所有类import java.util._

  4. import导入相同包的多个类，不必像Java那样写很多行import java.util.{List,Set}

  5. 给类起别名，如果别名为_的话，是忽略这个类

  6. import可以直接导入包

  7. 如果包名冲突，可以使用_root_通过全路径来找到自带类

类属性访问权限

public

public是默认的访问权限，但是Scala并没有public关键字

protected

访问权限只能子类访问，和Java不同；同包也不能访问

private

私有访问权限只能在类内部访问或者此类的伴生对象内访问

default

没有default关键字，通过private[包]设置能被访问到的包。

类的方法

类的方法和函数只有唯一的区别，方法的调用必须基于对象。

通过伴生对象来构建伴生类的对象

构造方法🔺

Scala的构造方法有两种：主构造方法和辅助构造方法。

  • Scala构造对象可以通过辅助构造方法创建，但是必须首先调用主构造方法
  • 主构造方法是声明类时就要指定参数，没有可以不写。class User、class User(s:String)
  • Java中构造方法名就是类的名字；但Scala中主构造方法直接在类名后写就行，辅助构造方法全是使用this关键字

类、继承、抽象

继承

和Java一样，使用extends；不可多继承

抽象类

abstract修饰，抽象类和普通类的区别 有：

  • 属性可以不必显式赋值，这时属性是抽象的
  • 抽象方法只需要声明即可，不必实现
  • 子类必须实现抽象父类的抽象方法，补全父类的方法即可。此时可加可不加overwrite关键字，IDEA会默认加上
  • 子类如果想重写父类方法/属性，必须加入overwrite关键字，不加入报错

Java的方法动态绑定🔺

父子类的重写方法执行，子类执行方法时

  • 首先查找子类内存中是否有此方法
  • 没有的话寻找父类是否由此方法，这时执行的是父类的方法
  • 参数没有动态绑定机制，他函数内的方法就是它类中的不是子类

父类的构造方法

如果父类的构造方法有参数，那么实现父类的子类必须显式的传递参数

trait特质

Scala没有接口的概念，有特质的概念(trait)，类似于接口

特质声明

特质可以继承

如果类继承多个特质，用with连接

如果类同时存在父类和特质，依赖采取继承方法，但继承的是父类，连接的是特质

特点

  1. 特质无法构建对象
  2. 特质可以执行代码
