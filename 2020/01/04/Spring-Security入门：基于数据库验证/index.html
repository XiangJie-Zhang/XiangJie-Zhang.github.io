最重要的配置

Spring Security的配置

个人习惯在设置接口角色时，不喜欢在配置类里一一设置，我比较喜欢在control层的方法上使用注解：

使用注解需要注意：

  1. 下面类的三个注解必须要加！！
  2. 如果使用注解@Secured("ROLE_USER")，需要在注解EnableGlobalMethodSecurity加上参数securedEnabled=true
  3. 如果使用注解 @PreAuthorize("hasRole('ADMIN')")，需要在注解EnableGlobalMethodSecurity加上参数prePostEnabled=true

上述两个注解都是指定角色才可以访问，但是有个大坑！！

  1. @PreAuthorize("hasRole('ADMIN')")由于使用了hasRole方法，它所匹配的角色为ROLE_ADMIN，他会给你加个前缀，所以你数据库存储的角色也必须叫ROLE_ADMIN;Spring Security所有的hasROle方法都会给你加ROLE_前缀
  2. @Secured("ROLE_USER")这里就直接匹配你数据库里面的角色。

使用jwt代替session

  1. 登陆成功后返回token数据，使用上面的工具类生成，同时修改配置，取消自带的session,sessionManagement().sessionCreationPolicy(SessionCreationPolicy.STATELESS)
  2. 根据自己需要修改token工具类
  3. 设置filter，解析我们自己的token
  4. 把filter添加到配置中，http.addFilterBefore(tokenAuthorizationFilter,UsernamePasswordAuthenticationFilter.class);
